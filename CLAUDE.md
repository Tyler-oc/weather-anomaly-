## Rules

1. Always write unit tests before writing code to solve the unit tests. Ensure the unit tests are comprehensive to the feature being developed.

2. Functions should have only one purpose and functionality. If a function becomes difficult to explain, it is too long and should be broken up into multiple functions.

3. Helper functions are preferred for resusable code. Always check if the code you are writing is something that will likely be reused and write a helper function if so.

4. Always update ARCHITECTURE.md with any changes made. This is the source of truth for the directory layout. Please write it in a way that is easy for you to navigate later on to find files and understand the codebase structure.
